# Delivery & Release Rules — Cynthia Design Studio
# Version: 1.0.0

## Delivery Gating

Nothing leaves the studio without passing ALL of these gates:

### Gate 1: Quality Score
- Overall UDEC score ≥ 8.5
- No blocker axis (MOT, ACC) below 7.0
- Score must be logged in audits/

### Gate 2: Production Checklist
Every deliverable passes the 20-point production checklist:
1. Zero console errors
2. Zero broken links
3. Zero placeholder content (Lorem, TODO, Coming Soon)
4. All images optimized and have alt text
5. All forms have validation and accessible labels
6. All interactive elements have :hover, :focus, :active states
7. All animations respect prefers-reduced-motion
8. Mobile layout tested at 375px
9. Touch targets ≥ 44px on mobile
10. Lighthouse Performance ≥ 90
11. Lighthouse Accessibility ≥ 90
12. No horizontal overflow at any breakpoint
13. Favicon and meta tags present
14. Semantic HTML structure (no div soup)
15. No inline styles (except generated by JS animation libraries)
16. All fonts loaded via font-display: swap
17. No render-blocking resources
18. All images lazy-loaded below fold
19. Color contrast passes WCAG AA
20. CTA copy is specific and actionable (no "Click Here" or "Learn More")

### Gate 3: Human Approval (when required)
Per APPROVAL_BOUNDARIES.md:
- Client-facing deliverables require human approval
- Internal improvements can ship autonomously
- Financial actions always require human approval

## Delivery Formats

### Web Deliverables
- Single HTML file (self-contained, all CSS/JS inline) for simple pages
- Next.js build output for complex applications
- Both versions if applicable

### Design Assets
- Source files (Figma link or exported)
- Production-ready images (WebP + fallback JPG/PNG)
- Icons (SVG, optimized)

### Video Deliverables
- MP4 (H.264, AAC audio)
- Multiple aspect ratios: 16:9, 9:16, 1:1
- Thumbnail frame
- Subtitle file (SRT) if applicable

### Audio Deliverables
- MP3 (320kbps)
- WAV (uncompressed, for editing)
- Transcript text file

### Reports & Documents
- PDF (branded template)
- Markdown (for web display)
- JSON (for programmatic consumption)

## Delivery Channels
- Client Portal (primary)
- Email with secure download link
- Cloudflare Pages deployment (websites)
- Direct file delivery (for enterprise)

## Post-Delivery
1. Move job to studio/jobs/completed/
2. Write project memory with lessons learned
3. Update Studio Memory with new insights
4. Generate invoice (if billable)
5. Schedule followup (14 days) for feedback collection
